# Keka Authentication

This guide walks you through setting up the Keka integration within Boltic.

## Prerequisites

To set up the Keka integration, you'll need:

-   An keka account (Client ID, Client Secret, API Key)
-   Following is he link to know more about client id, client secret and api key and how to use them to create new access token <a href='https://developers.keka.com/reference/authentication' target='_blank' rel="noopener">Authentication Documentation</a>.

If you don't have an Keka account, you can create one at <a href="https://www.keka.com/signup" target="_blank" rel="noopener"><span>Keka Create Account</span></a>.

## Setting Up the Keka Integration in Boltic

1. **Log In**

    - Log into your Keka account.

2. **Log into Boltic**

    - Log into your Boltic account or navigate to the Boltic Open Source dashboard.

3. **Navigate to Integrations**

    - Click on `Sources` and then click on `+ Add New source`.

4. **Select Source Type**

    - On the Set up the source page, search for `Keka` from the search bar and select it.

5. **Basic Details**

    - **Name:** Enter a name for the Keka integration.

    - **Description:** Provide a brief description of the integration.

6. **Connection Details**

    - **Authentication Type:** Select authentication type.

    - **Company:** Provide the company name.

    - **Environment:** Select the environment you want to use.

    - **Client ID:** Provide the client id.

    - **Client Secret:** Provide the client secret.

    - **API Key:** Provide the api key.

7. **Test and Save Integration**

    - Click `Test Connection` to ensure the connection is successful.
