"# **OpenRouter API Authentication**\n\nGuide on authenticating API requests for the OpenRouter AI API.\n\nThis document provides a comprehensive guide on how to authenticate API requests for the OpenRouter API. The system uses **API Keys** to ensure secure communication with OpenRouter’s model routing platform.\n\n---\n\n## **Authentication Types**\n\nOpenRouter API supports the following authentication type:\n\n-   **API Key**: A secure token that identifies and authorizes your application to access OpenRouter’s supported language models.\n\n---\n\n### **Obtaining Your API Key**\n\n1. Log in to the <a href=\"https://openrouter.ai/settings\" target=\"_blank\">OpenRouter Dashboard</a>.\n2. Scroll to the **API Key** section.\n3. Click **\"Create Key\"** to generate a new key, or copy an existing one.\n4. Store this key securely. It starts with `sk-or-`.\n\n---\n\n## **Validation**\n\nTo validate the key, make a test request to the endpoint using **Test & Save** button.\nA successful response confirms your key is valid and active.\n\n---\n\n## **Troubleshooting**\n\nIf authentication fails:\n\n-   Ensure your **API Key** is copied correctly.\n-   Confirm the key begins with `sk-or-`.\n-   Check that the key has not been **revoked** or **expired**.\n"
