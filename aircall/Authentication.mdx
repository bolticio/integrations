"# Aircall Authentication\n\nThis document outlines the setup for Aircall authentication in Boltic, using an API key. The authentication method can be selected and configured through the following steps.\n\n## Authentication Types\n\n### 1. API Key Authentication\n\nTo authenticate with Aircall, you can use the **API Key** method. Follow the steps below to configure the authentication.\n\n### Parameters\n\n#### Authentication Type\n\n-   **Parameter Name:** `type`\n-   **Display Name:** Authentication Type\n-   **Description:** Choose the type of authentication you want to use.\n-   **Type:** Select\n-   **Options:**\n    -   **API Key:** Use your Aircall API key for authentication.\n-   **Validation:** Required\n\n#### API Key Authentication Fields\n\nWhen `API Key` is selected as the authentication type, the following fields are required:\n\n##### API ID\n\n-   **Parameter Name:** `api_id`\n-   **Display Name:** API ID\n-   **Description:** Your Aircall API ID.\n-   **Type:** Text\n-   **Placeholder:** Enter API ID\n-   **Validation:** Required\n\n##### API Token\n\n-   **Parameter Name:** `api_token`\n-   **Display Name:** API Token\n-   **Description:** Your Aircall API Token.\n-   **Type:** Password\n-   **Placeholder:** Enter API Token\n-   **Validation:** Required\n\n### Validation\n\nOnce the API ID and API Token are entered, the connection is validated using the Aircall API.\n\n-   **Validation URL:** `https://api.aircall.io/v1/ping`\n-   **Method:** GET\n-   **Headers:**\n    -   **Accept:** `application/json`\n    -   **Authorization:** Basic authentication using the API ID and API Token\n-   **Response:**\n    -   **Success:** The response will include output from `{{response.data}}`.\n    -   **Error:** If an error occurs, the message will be displayed as `{{response.data.message}}`, with the corresponding status code: `{{response.status}}`.\n\n## Troubleshooting\n\nIf the connection fails:\n\n-   Ensure your **API ID** and **API Token** are correct.\n-   Confirm that your Aircall account has the necessary permissions for API access.\n-   Verify the network connection.\n\n## Conclusion\n\nAfter successfully setting up your API key, you should be able to authenticate and start interacting with the Aircall API through Boltic.\n"
